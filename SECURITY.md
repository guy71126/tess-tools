# Security Policy

## Supported versions

Until a stable release exists, security fixes are applied to the latest published version and the `main` branch.

## Reporting a vulnerability

Do not open a public issue for a vulnerability involving credential exposure, unsafe file handling, or unintended network access. Use GitHub's private vulnerability reporting feature for this repository. Include reproduction steps, affected commands, and the expected impact without attaching sensitive credentials or private datasets.

Ordinary provider failures, malformed public archive responses, and incorrect product classifications should use the public issue templates.
